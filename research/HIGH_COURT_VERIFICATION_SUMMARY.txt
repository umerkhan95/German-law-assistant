PESHAWAR HIGH COURT (PHC) - TIER 1

Entry Points:
  1. https://www.peshawarhighcourt.gov.pk/PHCCMS/reportedJudgments.php
  2. https://peshawarhcatd.gov.pk/search_cases.phc (Abbottabad)
  3. https://www.peshawarhcmb.gov.pk/ (Mingora)
  4. https://diary.peshawarhighcourt.gov.pk/ (Case diary)

Technology Stack:
  • Frontend: jQuery DataTables
  • Response: HTML pagination (25 records/page)
  • Rate Limit: None (2-5 req/sec OK)
  • CAPTCHA: None
  • Session: Standard HTTP cookies
  • PDFs: Direct URLs (/PHCCMS/judgments/[CASE_REF].pdf)

Crawling Strategy:
  1. Extract filter values (years 2010-2026, judges, categories)
  2. Iterate (year, judge, category) combinations
  3. Request with filters, parse DataTables response
  4. Download PDFs via direct URLs
  5. Extract metadata from PDF text

Implementation Time: 1 week
Maintenance: Low (monitor for structural changes)
Proxy Needed: No
CAPTCHA Solver: No


ISLAMABAD HIGH COURT (IHC) - TIER 2

Entry Points:
  1. https://mis.ihc.gov.pk/frmCseSrch (case search)
  2. https://mis.ihc.gov.pk/frmSrchOrdr (order/law search)
  3. https://mis.ihc.gov.pk/frmLibDtl (library - optional)

Technology Stack:
  • Backend: ASP.NET with AJAX
  • Case Search: JSON AJAX responses (srchCseIhc_ByInst method)
  • Order Search: HTML card layouts
  • Rate Limit: Unknown (start with 1-2 req/sec, monitor)
  • CAPTCHA: Unknown (integrate CapSolver if appears)
  • Session: ASP.NET session cookies required
  • PDFs: Direct URLs (/attachments/judgements/[ID]/[FILE]/[NAME].pdf)
  • Main Site: AVOID (timeouts on ihc.gov.pk) - use MIS only

Special Handling:
  • Exponential backoff for timeouts (1s → 5s → 30s)
  • Session persistence across requests
  • Timeout set to 30 seconds
  • Monitor for HTTP 429/503 responses
  • Enable crawl4ai stealth mode

Case Types Supported:
  • W.P. (Writ Petition)
  • I.C.A. (Islamabad Court Appeal)
  • I.T.R. (Income Tax Review)
  • Crl. Misc. (Criminal Miscellaneous)
  • S.T.R. (Sales Tax Review)
  • Cust. Ref. (Customs Reference)
  • F.A.O. (First Appeal in Order)
  • O.W.P. (Original Writ Petition)

Crawling Strategy:
  1. Enumerate case types + years (2000-2026)
  2. Search by case number (iterate 1-9999 per type/year)
  3. Parse AJAX JSON responses
  4. Extract metadata + PDF links
  5. Download PDFs with timeout retry
  6. Alternative: Search by judge + citation for case law

Implementation Time: 2 weeks
Maintenance: Medium (monitor timeouts, rate limits)
Proxy Needed: No (but test with proxy if rate limited)
CAPTCHA Solver: Maybe (add CapSolver integration)
